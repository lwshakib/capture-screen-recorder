# Contributing to Capture Screen Recorder

Thank you for your interest in contributing to the Capture Screen Recorder project! We welcome contributions of all kinds, including bug fixes, new features, documentation improvements, and feedback.

## How to Contribute

1.  **Fork the repository**: Create your own copy of the repository.
2.  **Create a branch**: Use a descriptive name for your branch (e.g., `fix/video-sync-issue`).
3.  **Make changes**: Implement your changes or additions.
4.  **Test your changes**: Ensure everything works as expected across the different components (Web, Desktop, Extension).
5.  **Commit your changes**: Write clear and concise commit messages.
6.  **Submit a pull request**: Describe your changes and why you made them.

## Development Setup

The project is structured as a monorepo containing three main components:
- `capture-web`: The web application for hosting and managing recordings.
- `capture-desktop`: The Electron-based desktop application.
- `capture-extension`: The browser extension for quick recordings.

Refer to the individual `README.md` files in each directory for specific setup instructions.

## Coding Standards

- Use TypeScript for all JavaScript/TypeScript code.
- Follow the existing code style (Prettier/ESLint configurations are provided in most sub-projects).
- Write descriptive variable and function names.
- Keep components small and reusable.

## Reporting Issues

If you find a bug or have a feature request, please search existing issues before creating a new one. Provide as much detail as possible, including steps to reproduce the issue and your environment details.

## Questions?

If you have any questions, feel free to reach out or open an issue for discussion.

Happy coding!
